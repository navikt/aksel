# Ta i bruk `@navikt/ds-react` pakken

## Oppsett

```bash
yarn add @navikt/ds-react @navikt/ds-css
```

Siden stilene ikke importeres direkte i komponentene, så må man selv importere ds-css.
For å legge til all stylingen som trengs må man da bare importere `ds-css`.

```bash
import "@navikt/ds-css"
```

Dette vil også legge css-variabler på :root. For å få dette til å fungere med IE11, kan denne pakken tas i bruk: https://jhildenbiddle.github.io/css-vars-ponyfill/#/

Note: Disse komponentene kan ikke garanteres at fungerer som forventet i IE11 da vi har valgt og ikke unngå styling-valg basert på IE11 støtte eller ikke.

## Bruke styled-components eller emotion

Flere av våre komponenter er implementert med OverridableComponent. Dette vil si at komponenten tar i mot en prop, `as` som lar deg endre hvilken tag som blir satt.
Bruk for dette er eks hvis du ønsker `Heading` stilen, men ikke en heading-tag. Da kan man lett skrive `<Heading as="div">` og man vil få en `div`-tag istedenfor.

Flere css-in-js bibliotek har også denne funksjonaliteten bygd inne, noe som kan komplisere ting når alle bruker samme navn `as` for dette.
